## EKS 프로덕션 환경 작업 계획서 작성

### 변경 사항

- 변경된 파일: `docs/project-plan.md`
- 변경 내용: AWS EKS 프로덕션 환경 구축을 위한 상세 작업 계획서 작성
- 이유: 체계적인 프로젝트 진행을 위한 로드맵 필요

### 주요 내용

- 멀티 어카운트 구조 설계 (Organizations 확장 대비)
- 7개 Phase로 구분된 작업 계획
  - Phase 1: 기반 인프라 (State Backend, IAM)
  - Phase 2: 네트워크 (VPC, Subnet, NAT)
  - Phase 3: EKS 클러스터 (Control Plane, Node Groups, Add-ons)
  - Phase 4: 데이터 계층 (RDS Aurora, ElastiCache)
  - Phase 5: 보안 강화 (IAM, RBAC, Secrets)
  - Phase 6: 모니터링/로깅
  - Phase 7: CI/CD
- 프로덕션 베스트 프랙티스 체크리스트
- 최종 디렉토리 구조 정의

### 영향도

- 모든 후속 작업의 기준 문서
- 작업 우선순위 및 의존성 명확화

## 아키텍처 문서 Mermaid 다이어그램으로 변환

### 변경 사항

- 변경된 파일: `docs/architecture.md`
- 변경 내용: 모든 아키텍처 다이어그램을 Mermaid 형식으로 변환
- 이유: 시각적 가독성 향상 및 GitHub/IDE 미리보기 지원

### 추가된 다이어그램

- 인프라 개요: VPC, EKS, 데이터베이스, AWS 서비스 연결도
- 멀티 어카운트 구조: AWS Organizations 기반 계정 분리
- 레이어 배포 의존성: 10개 레이어 간 의존 관계 (컬러 코딩)
- VPC 구성도: AZ별 서브넷 배치 (Public/Private/Database/Pod)
- EKS 클러스터 구성: Control Plane, Node Groups, Add-ons
- State 관리: S3 + DynamoDB 백엔드
- IAM 구조: Cluster Roles, IRSA, User Roles
- 네트워크 보안: Security Group 체인
- 고가용성 설계: Multi-AZ 구성

### 영향도

- 문서 시각화 품질 향상
- GitHub, VSCode에서 다이어그램 렌더링 지원
- 아키텍처 이해도 향상
